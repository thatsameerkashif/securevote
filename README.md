Secure Voting App is a full-stack web application designed to facilitate transparent, tamper-resistant online elections. Built with PHP and MySQL, the system provides a complete end-to-end voting experience — from voter registration and authentication to real-time result tracking by administrators.
The application supports two distinct user roles. Voters can create accounts, log in securely, browse active elections, and cast their vote for a candidate of their choice. The system enforces a strict one-vote-per-election rule at the database level using foreign key constraints, ensuring no voter can submit duplicate votes. Administrators have access to a dedicated dashboard where they can create new elections, add candidates, monitor live vote counts with visual progress bars, toggle elections between active and closed states, and permanently delete elections when needed.
Passwords are protected using bcrypt hashing, meaning raw passwords are never stored in the database. All user sessions are managed securely with PHP's native session handling, and role-based access control ensures voters cannot access admin pages and vice versa. Every voting action is logged through a built-in audit trail, providing a record of all activity within the system.
The frontend features a modern, responsive UI built entirely with custom CSS — no frameworks or libraries. The design uses a deep navy and electric indigo color palette, Space Grotesk and Inter typefaces, and fluid layouts that adapt cleanly across desktop, tablet, and mobile screen sizes.
The database schema is structured around four core tables — users, elections, candidates, and votes — with properly enforced foreign key relationships to maintain referential integrity throughout all operations.
This project demonstrates practical skills in relational database design, server-side scripting, secure authentication, role-based authorization, responsive UI design, and full-stack deployment on a live PHP hosting environment.
